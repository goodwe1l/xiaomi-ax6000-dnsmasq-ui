# 10.0.0.1 DHCP Web 管理页（8088）操作记录

## 1. 目标

在 `10.0.0.1` 上提供一个独立的 Web 管理页（`8088` 端口），用于可视化管理：

- LAN DHCP 开关
- 默认 DHCP 选项（Option 3/6）
- 静态租约（IP/MAC/设备名）
- 标签模板（Tag）
- 每设备按标签下发网关/DNS
- 动态租约查看与一键转静态

## 2. 环境与约束

- 设备：`10.0.0.1`（小米魔改 OpenWrt 18.06）
- 根分区：`squashfs` 只读，`/www` 不可写
- 可写持久目录：`/data`
- 因此采用独立 `uhttpd` 实例，不改原生 LuCI

## 3. 最终部署结构

### 3.1 路由器端（10.0.0.1）

- 程序目录：`/data/dhcp_adv/`
  - `cgi-bin/dhcp_adv_api`（Go JSON API CGI 二进制）
  - `web/index.html`（前端入口）
  - `web/assets/*`（CSS/JS/图标/前端依赖）
  - `start.sh`（启动 8088）
  - `ensure.sh`（存活检查）
- 站点目录：`/data/dhcp_adv/www/`（由 `start.sh` 维护）
- PID 文件：`/data/dhcp_adv/uhttpd.pid`
- 运行端口：`10.0.0.1:8088`
- 访问地址：`http://10.0.0.1:8088/`

### 3.2 本地目录（本机备份）

- 路径：`10.0.0.1_dhcp_web_8088/`
- 文件：
  - `cmd/dhcp_adv_api/main.go`
  - `cgi-bin/dhcp_adv_api`（本地编译产物）
  - `web/index.html`
  - `web/assets/css/material3.css`
  - `web/assets/js/app.js`
  - `web/assets/js/api.js`
  - `web/assets/icons/sprite.svg`
  - `web/assets/vendor/petite-vue.es.js`
  - `dhcp_adv_start.sh`
  - `dhcp_adv_ensure.sh`
  - `dhcp_adv_index.html`（提示页）
  - `操作记录.md`

## 4. 已实现功能

### 4.1 DHCP 控制

- 一键切换 `dhcp.lan.ignore`
- 已完成切换：
  - `10.0.0.2` DHCP 关闭
  - `10.0.0.1` DHCP 开启

### 4.2 默认规则

页面可配置 `dhcp.lan.dhcp_option`：

- Option 3（网关）
- Option 6（DNS）

当前已配置：

- `3,10.0.0.1`
- `6,222.246.129.80,59.51.78.210`

### 4.3 标签模板

支持：新增/更新/删除（删除时有“被静态租约占用”保护）

当前模板：

- `istoreos` -> `3,10.0.0.2` / `6,10.0.0.2`
- `8` -> `3,10.0.0.8` / `6,10.0.0.8`

### 4.4 静态租约

支持：新增、编辑、删除、按标签绑定。

关键修复：

- 该系统 `dnsmasq` 需按 **tag section 名** 匹配（如 `tag_istoreos_`），
  不是直接按显示名 `istoreos`。
- 已修复映射逻辑，避免“页面显示有标签但下发不生效”。

### 4.5 动态租约

- 按网段分为两块：
  - `动态租约（LAN 当前已分配）`
  - `其他网段动态租约（如 MIoT）`
- 每条支持“转为静态”（预填表单）
- 已改为：**动态列表包含静态设备并标记类型**
  - 类型：`静态保留` / `动态`

## 5. 自恢复与持久化

已配置：

- `* * * * * /data/dhcp_adv/ensure.sh`
- `@reboot /data/dhcp_adv/start.sh`

说明：

- 重启后会自动拉起页面
- 若 `uhttpd:8088` 意外退出，最多 1 分钟内恢复

## 6. 常用维护命令

### 6.1 查看 8088 进程

```sh
ps w | grep '[u]httpd' | grep '10.0.0.1:8088'
```

### 6.2 手动重启页面

```sh
/data/dhcp_adv/start.sh
```

### 6.3 查看租约

```sh
cat /tmp/dhcp.leases
```

### 6.4 查看 DHCP 关键配置

```sh
uci -q show dhcp.lan
uci -q show dhcp | grep '=host\|=tag'
```

### 6.5 一键部署（本地执行）

```sh
cd 10.0.0.1_dhcp_web_8088
ROUTER_PASS='路由器SSH密码' ./deploy_oneclick.sh --host 10.0.0.1
```

相关文档：

- `httpd配置与一键部署说明.md`

## 7. 备注

- `192.168.32.x` 租约来自 `miot` 接口，不是 LAN。
- 如果某设备在动态区没出现，先查 `/tmp/dhcp.leases` 是否有该 MAC。
- 若页面打不开，先执行 `/data/dhcp_adv/start.sh` 再复测。
